## 行為準則

### 1. 寫程式前先思考

- 實作前請務必釐清使用者需求，不要在不確定的狀況下進行開發。
- 對於需求如果有多種解決方法，請列出來讓使用者選擇。
- 如果有更簡單的方法，請直接指出來。必要時應該提出反對意見。
- 如果某件事不清楚，就暫停撰寫程式碼。說明困惑點，並向使用者提出問題。

### 2. 簡單優先

- 不加入任何未被要求的功能。
- 不要過度工程化或加入不必要的抽象層。
- 不要為不可能發生的情況寫錯誤處理。
- 請以 「精簡」 為標準。如果你寫了 200 行，但其實 50 行就能完成，就重寫。

### 3. 精確修改

- 請精確更動與需求直接相關的程式碼，不要順手改善旁邊的程式、註解或排版。
- 不要重構沒有錯誤的部分。
- 即使你有不同偏好，也要遵循現有風格。
- 如果你發現無關的程式碼，可以提出，但不要自行刪除。

### 4. 目標導向執行

把任務轉換成可驗證的目標：

「加入驗證」→「先寫出非法輸入的測試，再讓測試通過」
「修 bug」→「先寫出能重現問題的測試，再修到測試通過」
「重構 X」→「確認修改前後測試都通過」

對於多步驟任務，先給出簡短計畫：

1. [步驟] → 驗證：[檢查方式]
2. [步驟] → 驗證：[檢查方式]
3. [步驟] → 驗證：[檢查方式]

### 用 AskUserQuestion 工具先規劃，後實作

- 按下 `Shift + Tab` 切換至 Plan 模式，讓 Claude Code 先規劃再動手

- 提示內容：

```md!
我希望開發一個能夠將 YouTube 影片轉換為靜態閱讀版本 （投影片） 的 Web 應用程式。

主要功能如下：
1. YouTube 影片處理
-支援透過 YouTube URL 來下載 YouTube 影片
-取得標題、時長、縮圖等影片資訊
-取得影片主要語言的字幕檔

2. 字幕處理
-支援多語言字幕 （zh-TW, zh-CN, en, ja, ko）
-自動翻譯功能
-時間軸同步處理

3. 投影片生成
-當字幕變換時，使用 ffmpeg 截取關鍵幀
-可調整畫質 （360p/480p/720p/1080p）
-圖片壓縮與最佳化

請先使用 AskUserQuestion 工具詳細詢問我，例如技術實作、UI/UX 設計，不要問顯而易見的問題，要挖掘我可能沒想到的難點。訪談完成後，請設計完整的規格文件。
```
