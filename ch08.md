```md!
幫我完成以下項目：
1.把難度分為4級(簡單、中等、困難、專家)，分級依據將參考人類解題時使用高級技巧(X-wing、swordfish...等等)的頻率以及挖空格數多寡。請先告訴我你認為的X-wing與swordfish是什麼。

3.建立初始首頁。
4.新增返回上一步、暫停功能。
5.遊玩時自動存檔(實時紀錄解題時間)。
6.可自選背景風格。
7.新增遊玩紀錄、生涯總覽頁面。
8.新增每日挑戰，每天都有專屬題目。
```

## 8-3 建立 PRD 高效生成程式

- CLAUDE.md：

```md!
# Python 環境

- 使用 uv 管理 Python 環境
- 回覆請都使用繁體中文
```

- PRD.md 範本：

```md!
# 數獨遊戲
請開發一個Python桌面應用程式，功能為數獨遊戲。從主選單點選不同功能進入對應畫面，整體風格使用現代扁平 UI，各按鈕與線條使用圓角框。

需包含以下功能：

- 難度選擇    簡單 / 中等 / 困難 / 專家，分級依據將參考人類解題時使用高級技巧(X-wing、swordfish...等等)的頻率以及挖空格數多寡
- 計時器      遊戲開始自動計時，完成後停止
- 提示        每局 3 次，隨機填入一個正確格子（顯示為綠色）
- 錯誤標示    填錯立即顯示紅色，計算錯誤次數
- 筆記模式    點「筆記」按鈕或按 N 鍵切換，可在格子記錄候選數字
- 高亮顯示    選取格子時，同行/列/宮 及 相同數字 自動高亮
- 鍵盤操作    方向鍵移動、數字鍵輸入、Delete/Backspace 清除
- 題目生成    最小剩餘值啟發法 + 回溯搜尋，大幅減少等待時間
- 初始首頁    每日挑戰、難度選擇、紀錄入口、背景主題
- 返回上一步  ↩ 按鈕或 Ctrl+Z
- 暫停        ⏸ 按鈕或 Escape 鍵
- 自動存檔    每次填入 + 每秒計時均自動存檔，重開頁面可繼續
- 背景風格    多種背景風格，首頁🎨選擇
- 遊玩紀錄    歷史清單篩選
- 生涯總覽    各難度最佳時間、零錯誤次數
- 每日挑戰    每天固定同一道題（日期為種子），記錄連續天數。
```

- 提示內容

```md
完成 @PRD.md
```

### 驗收遊戲

- 執行程式

```md

```

## 8-4 使用 Subagents 分工

### 為 Subagent 設定角色

- 建立Subagent

```md
你是遊戲邏輯工程師，負責數獨題目的生成、解題驗證與難度分級。難度分級需參考人類解題時使用進階技巧（X-Wing、Swordfish）的頻率，以及挖空格數的多寡。題目生成請使用最小剩餘值啟發法與回溯搜尋，確保生成速度流暢不卡頓。
```
```md
你是介面工程師，負責遊戲畫面的排版、按鈕配置、鍵盤操作與高亮顯示等互動功能。操作邏輯需符合直覺，並與遊戲邏輯工程師產出的模組串接。
```
```md
你是資料工程師，負責自動存檔、遊玩紀錄、生涯總覽，以及每日挑戰的題目種子與連續天數記錄。存檔需於每次填入及每秒計時時自動觸發，確保重新開啟程式後可繼續上次進度。
```
```md
你是視覺設計師，負責整體美術風格、色彩主題切換、深色模式，以及色塊與邊框的精緻化。深色模式下需確保九宮格線條清晰可見，整體風格需現代、簡潔。
```

- 檢視agent狀態


```md!
---
name: grill-me
description: 針對使用者的計畫或設計進行全面深入的提問，直到雙方對專案有一致的理解，逐一確認每個開發決策及其相互影響。適用於使用者想在開發前把需求想清楚、希望 Claude Code 主動提問釐清專案細節、或提到「grill-me」時使用。
---

針對這個計畫的每一個面向對我進行深入提問，直到我們對專案有一致的理解為止。逐一走訪每個開發決策，依序釐清各項決策之間的相互影響。每個問題請同時提供你的建議答案。

每次只問一個問題。

如果某個問題可以透過查看現有程式碼來回答，請直接查看，而非詢問使用者。
```
