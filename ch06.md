## 6-2 建立 hook 來阻止 Claude Code 修改檔案

- 在專案中建立 hook
  - 輸入的提示內容：
  ```md
  我想建立一個應用在這個專案的 PreToolUse hook，效果是強制擋下所有檔案修改動作：

  - 當 Claude 使用 Write、Edit 這些編輯檔案工具，或是執行 PowerShell 或 Bash 指令時觸發
  - 若工具符合，就回傳退出代碼2，不要執行任何外部程式
  - 直接阻止 Claude Code 使用，不要有任何訊息文字
  ```

  - 測試範例：
  ```md
  > 幫我把 tasks.json 第一個任務的狀態改成【完成】
  ```

- 把 PreToolUse hook 換成 PostToolUse hook
  - 測試範例：
  ```md
  > 幫我把 tasks.json 第二個任務的狀態改成【未完成】
  ```

- 修改執行的退出代碼
  - 測試範例：
  ```md
  > 幫我把 tasks.json 第三個任務移除
  ```

- 透過標準錯誤輸出阻擋原因
  - 輸入的提示內容：
  ```md
  幫我修改 PreToolUse hook，在以退出代碼 2 阻擋工具前，先透過標準錯誤輸出說明阻擋原因：「禁止修改：此專案的 hook 已封鎖所有檔案編輯操作」
  ```
  - 測試範例：
  ```md
  > 幫我在 tasks.json 添加第四個任務【測試 hook 運作】
  ```

## 6-3 在 Telegram 上收到 Claude Code 的提醒通知

- 建立 Notification hook
  - 輸入的提示內容：
  ```md
  我想在這個專案中建立一個 Notification hook，效果是當 Claude Code 觸發通知事件時，會自動執行外部程式來處理通知：
	
  - 在 .claude/settings.json 中新增 Notification hook 設定
  - hook 觸發時執行 .claude/hooks/telegram_notification.py 外部程式
  - 我是用 uv 管理專案，所以請使用 uv 執行命令
  - 外部程式 telegram_notification.py 稍後才會建立，這個步驟只需要完成 hook
  ```
- 建立 Telegram 機器人並取得 API token 與 CHAT_ID
  - [Telegram 註冊步驟](https://hackmd.io/@flagmaker/S1Cp6uBlzx)
  
  - 取得 CHAT_ID 的網址格式：
  ```md
  https://api.telegram.org/bot<你的API token> /getUpdates
  ```

- hook 執行的命令 – 發送提醒通知到 Telegram
  - 輸入的提示內容：
  ```md
  我已經在這個專案中建立好一個 Notification hook，但telegram_notification.py這個程式還未設計，請依照我的要求設計出這個程式：

  我希望這個程式能夠接收 Claude Code 的通知事件，當 Claude Code 發送通知時，將通知訊息轉發到我的 Telegram 機器人。程式會根據傳入的 JSON 資料發送 Telegram 通知訊息，JSON資料格式請參考官方文件。

  程式需要：
  1. 只處理 hook_event_name 為 "Notification" 的事件
  2. 從 .env 檔案讀取 Bot Token 和 CHAT_ID
  3. 使用 Telegram Bot API 發送推播訊息
  4. 我是用 uv 工具來管理專案，所以請使用 uv 來操作。
  5. 建立 .env 檔案來儲存 Telegram 設定。
  ```

  - 設定 Telegram 的 Bot Token 與 CHAT_ID：
  ```md
  幫我修改 .env 中的 Telegram 相關設定，我的 Bot Token 是 XXXXX，CHAT_ID 是 XXXXX
  ```
- 測試hook有沒有順利運作
  - 測試範例：
  ```md
  > 幫我在 tasks.json 添加一個新的任務【測試 Telegram 通知】狀態設定為【未完成】
  ```
  
## 6-4 讓 Hook 更加靈活：JSON 輸出介面

- 用 JSON 輸出介面取代退出代碼
  - 輸入的提示內容：
  ```md
  幫我把這個專案的 PreToolUse hook 改寫，從退出代碼改為使用 Claude Code 提供的 JSON：

  - 不要修改原有的觸發時機 PreToolUse 與觸發條件  matcher
  - 請先參考 Claude Code 的官方文件，再來設定命令的 JSON 輸出介面：https://docs.anthropic.com/en/docs/claude-code/hooks
  - command 改成使用 JSON 輸出介面並設定為 "deny" 來拒絕 Claude Code 使用檔案編輯工具
  - 拒絕的同時也要加上拒絕原因："這個專案不允許檔案被修改"
  - 保持不執行任何外部程式
  ```
  - 測試範例：
  ```md
  > 幫我在 tasks.json 添加一個新的任務【了解 JSON 輸出介面】
  ```

- JSON："ask" 強制經過允許才能操作
  - 輸入的提示內容：
  ```md
  我想要在 Claude Code 每次使用編輯檔案的工具前，都需要我的同意才能繼續操作，所以幫我把這個專案的 PreToolUse hook 之中的 permissionDecision 改成 "ask"
  ```
  - 測試範例：
  ```md
  > 幫我在 tasks.json 的最後一個任務刪除
  ```

- JSON："updatedInput" 強制修改操作內容
  - 輸入的提示內容：
  ```md
  幫我在這個專案中建立一個新的 PreToolUse hook，在讀取檔案工具執行前修改要讀取的檔案：

  - 當 Claude Code 使用 Read 的讀取檔案工具時觸發
  - 請先參考 Claude Code 的官方文件，再來設定命令的 JSON 輸出介面：https://docs.anthropic.com/en/docs/claude-code/hooks
  - 命令使用 JSON 輸出介面並設定為 "updatedInput" 來修改要讀取的檔案
  - 若工具符合，就將要讀取的檔案都改成讀取 tasks.json 這個檔案 
  - 保持不執行任何外部程式
  ```
  - 測試範例：
  ```md
  > 我想看看現在有什麼 hook，幫我讀取設定檔 settings.json
  ```

## 6-5 讓 Hook 條件更加精準

- 用 if 條件式篩選檔案
  - 輸入的提示內容：
  ```md
  幫我修改這個專案的 PreToolUse hook，讓「詢問」只在修改特定檔案時觸發：

  - 請先參考 Claude Code 的官方文件：https://docs.anthropic.com/en/docs/claude-code/hooks
  - 使用官方的 if 條件語法篩選，分別為 Write 與 Edit 各設一條規則，只有操作目標符合 "tasks.json" 的才觸發
  - 若符合，使用 JSON 輸出介徵詢使用者同意
  - command 只需用 echo 輸出 ask JSON，不需要任何條件判斷邏輯
  - 不要執行任何外部程式
  ```
  - 測試範例：
  ```md
  > 幫我刪除 tasks.json 的第一個任務
  ```

- 利用 JSON 輸入介面篩選檔案
  - 輸入的提示內容：
  ```md
  幫我修改這個專案的 PreToolUse hook，讓「詢問」只在修改特定檔案時觸發：

  - 保持觸發時機 PreToolUse 與 matcher 不變
  - 請先參考 Claude Code 的官方文件：https://docs.anthropic.com/en/docs/claude-code/hooks
  - 將 command 改為執行一個程式腳本，腳本從標準輸入讀取 Claude Code 傳入的 JSON 物件，判斷 tool_input.file_path 是否包含 "tasks.json"
  - 如果是 tasks.json，輸出 JSON 輸出介面並設定 permissionDecision 為 "ask"
  - 如果不是，輸出 JSON 輸出介面並設定 permissionDecision 為 "allow"
  - 腳本請用 Python 撰寫，存放到 .claude/hooks/ 目錄下
  - 我用 uv 管理專案，請用 uv 執行腳本
  ```
  - 測試受保護範例：
  ```md
  > 幫我在 tasks.json 添加一個新任務【追蹤特定檔案】
  ```
  - 測試未受保護範例：
  ```md
  > 幫我在 todo.py 的最上方加入一行註解【追蹤特定檔案】
  ```

- 用 if 篩選 Bash/PowerShell 命令
  - 輸入的提示內容：
  ```md
  請幫我在這個專案中建立一個 PreToolUse hook，攔截包含危險指令的 Bash 操作：

  - 請先參考 Claude Code 的官方文件：https://docs.anthropic.com/en/docs/claude-code/hooks
  - 使用官方的 Bash 指令比對語法（Bash(pattern)）分別攔截以 rm、sudo、chmod 開頭的危險Bash 指令
  - 若符合，使用 JSON 輸出介面 hookSpecificOutput 的 deny 阻擋，並附上拒絕原因
  - command 只需用 echo 輸出 deny JSON，不需要任何條件判斷邏輯
  - 不要執行任何外部程式
  ```
  - 測試封鎖範例：
  ```md
  > 幫我用 Bash 指令刪除 delete.txt
  ```
  - 測試未封鎖範例：
  ```md
  > 幫我用 Bash 指令建立一個檔案 add.txt
  ```

- 用外部程式篩選 Bash/PowerShell 命令
  - 輸入的提示內容：
  ```md
  幫我把剛剛的 hook 改成執行外部程式來判斷，並依據指令的危險程度給出不同回應：

  - 用 Python 設計一個判斷 Bash 指令的程式
  - 程式從 hook 傳入的 JSON，取出要執行的 Bash 指令
  - 分別建立「高風險」與「低風險」指令清單，供程式比對並判斷：
    - 高風險指令（rm、sudo）回傳 deny 直接阻擋
    - 中風險指令（curl、mv）回傳 ask 讓使用者自行判斷
  - hook 的命令使用 uv 執行這個腳本
  ```
  - 測試刪除範例：
  ```md
  > 幫我用 Bash 指令把 move.txt 搬移到 data 資料夾中
  ```
  - 測試允許範例：
  ```md
  > 幫我用 Bash 指令把 move.txt 刪除
  ```

- 讓 Claude 模型檢查與篩選
  - 輸入的提示內容：
  ```md
  再次修改專案中現有的 PreToolUse hook，作用是讓 Claude 模型判斷該不該允許這個Bash 指令，需求如下：

  - matcher 欄位設定為 Bash，只攔截 Bash 指令
  - type欄位使用 prompt 處理類型，將指令交給 Claude 模型判斷安全性
  - prompt 欄位先留空，後續再補上
  - 我想使用 Claude 最新的 Opus 4.7 模型來幫我判斷
  - timeout 設為 15 秒
  ```
  - 生成 prompt 內容：
  ```md
  請幫我生成一段適合填入 prompt 欄位的內容，讓模型能分析 Bash 指令的安全性，並給出允許或拒絕的回覆，其中如果有關重要檔案例如 .env，請直接給出拒絕的回覆
  ```
  - 測試阻止範例：
  ```md
  > 幫我用 Bash 查看 .env 檔的內容
  ```
  - 測試未阻止範例：
  ```md
  > 幫我用 Bash 查看 /src/todo.py 的內容
  ```
